Build a web application for **multi-agent market intelligence** focused on **forex, gold, and oil**. The platform should analyze markets using multiple specialized agents, aggregate their outputs, and present explainable analytics, confidence scores, reports, and alerts. The product is not just a signal tool. It should feel like an **institutional-style market intelligence dashboard**.

## Product goal

Create a subscription-based web app where users can:

* view market analysis for forex pairs, gold, and oil
* see multiple AI/quant agents analyze the same market
* view consensus, confidence, key levels, and invalidation
* compare where agents agree and disagree
* receive alerts and read daily/intraday reports
* unlock more markets, agents, and features depending on plan

The key differentiator is **cross-market dependency analysis**. Markets should not be analyzed in isolation. Example: USD/CAD should incorporate crude oil and dollar strength. Gold should incorporate dollar strength and real yields. USD/JPY should incorporate yields and risk sentiment.

## Core product concept

The system should have multiple layers:

1. **Independent market agents**

   * each market gets analyzed by specialized agents

2. **Cross-market dependency agent**

   * reads outputs from related markets and drivers
   * determines whether related markets are confirming or conflicting with the target market

3. **Regime detection**

   * determines if relationships are valid in the current regime
   * examples: trending, ranging, risk-off, event-driven, panic, carry-friendly

4. **Meta-consensus agent**

   * combines all independent agent outputs plus dependency outputs
   * returns a final market view

## Most important architectural rule

Do **not** rely purely on LLM opinion. Use deterministic analytics first and LLMs second.

* deterministic/statistical logic should produce the real scores and signals
* LLMs should mostly be used for explanation, summaries, report writing, and Q&A
* every agent should return structured JSON
* every output should be stored for history, evaluation, and backtesting

## Cross-market scoring behavior

For each dependency, classify it as:

* strongly_confirming
* confirming
* neutral
* conflicting
* strongly_conflicting

Example:
If USD/CAD base bias is bullish:

* DXY bullish = confirming
* WTI bearish = confirming
* rate spread bullish USD = confirming

Then the dependency agent should boost confidence.

If gold base bias is bullish:

* DXY bullish = conflicting
* real yields rising = strongly conflicting
* geopolitical stress high = confirming

Then confidence should be reduced or mixed.

## Key implementation rules

* build the system so every agent output is auditable
* store all predictions and confidence values historically
* make dependency logic config-driven
* make agent outputs structured first, natural language second
* separate market-specific logic from shared orchestration logic
* do not hardcode simplistic permanent assumptions like "oil down always means USD up"
* support regime-aware relationships
* surface disagreements instead of hiding them
* prioritize explainability in the UI
